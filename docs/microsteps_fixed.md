## PATH: методичка по микро-шагам

Этот документ описывает контентную библиотеку микро-шагов для всех 7 стадий PATH. Он нужен как редакторская и продуктовая опора: для наполнения `HOME`, развития AI-подсказок, ручной кураторской работы и будущих сценариев персонализации.

## Как пользоваться методичкой

- Выбирай микро-шаг по текущей стадии PATH, а не по желанию "сделать всё сразу".
- Один микро-шаг должен вести к одному наблюдаемому действию за короткую сессию.
- Не проходи список линейно от `1` до `100`: это библиотека вариантов, а не обязательный чек-лист.
- Если формулировка слишком широкая, сужай её до действия на 5–20 минут.
- После выполнения фиксируй результат: демо, заметку, решение, публикацию, метрику или следующий шаг.

## Связь с runtime

- Боевой источник микро-шагов для приложения: `src/lib/micro-step-prompts.ts`.
- API, который отдаёт микро-шаг в `HOME`: `src/app/api/home/micro-step/route.ts`.
- Эта методичка не подключается к runtime напрямую и служит расширенной контентной базой.
- Если меняется пользовательский контент в приложении, обновляй и `src/lib/micro-step-prompts.ts`, и этот документ, чтобы редакторская база не расходилась с продуктом.

## Принцип составления микро-шагов

- Формулировка начинается с глагола действия.
- Один пункт покрывает одну мысль, без скрытых подзадач.
- Шаг должен помогать двигаться по стадии, а не просто создавать ощущение занятости.
- Внутри каждой стадии есть смесь творческих, организационных, коммуникационных и стратегических действий.
